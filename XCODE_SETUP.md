# Xcode Setup Guide for ScanToExternApp (macOS)

This guide gets you a **real, buildable, runnable .app** from the source tree in this repo.

The sources are structured for easy import into a standard Xcode macOS App project.

## Step-by-Step (macOS 13+ required, Xcode 15+ recommended)

1. **Create a new Xcode project**
   - Xcode → File → New → Project…
   - Choose **macOS** → **App**
   - Product Name: `ScanToExternApp`
   - Organization Identifier: `com.topscan`
   - Bundle Identifier will become `com.topscan.ScanToExternApp` (keep it — important for migration from v4)
   - Interface: **SwiftUI**
   - Language: **Swift**
   - **Uncheck** "Include Tests", "Use Core Data", "Host in CloudKit" (you can add tests later)
   - Create the project somewhere (e.g. next to this folder or inside `NewApp/mac`).

2. **Set deployment target**
   - In the project navigator, select the project (top level).
   - Select the **ScanToExternApp** target.
   - **Minimum Deployments** → macOS: **13.0** (Ventura).

3. **Replace / merge source files**
   - Delete the default `ContentView.swift` (and any generated `App.swift` if you want clean).
   - From Finder, drag the entire folder `mac/ScanToExternApp/` (or its contents: App/, MenuBar/, Hardware/, Injection/, OCR/, AI/, Preview/, History/, Settings/, Permissions/, Resources/) into your Xcode project.
   - When prompted:
     - Check **"Copy items if needed"** (recommended)
     - Add to target: **ScanToExternApp** (your app target)
   - Your project navigator should now have the folders under the app.

4. **Add the Resources (Info.plist + Assets)**
   - The `Resources/` folder we provide already has `Info.plist` and `Assets.xcassets`.
   - In Xcode target settings:
     - **Info** tab → "Custom macOS Application Target Properties"
     - Or easier: select the `Info.plist` we provided, set it as the "Info.plist File" in Build Settings (search "Info.plist").
   - Make sure `Assets.xcassets` is added to the target (it should be if you dragged the Resources folder).
   - **Critical plist keys** (they are in the provided `mac/ScanToExternApp/Resources/Info.plist`):
     - `LSUIElement` = `YES` (hides from Dock)
     - `NSBluetoothAlwaysUsageDescription`
     - `NSAccessibilityUsageDescription`
     - `com.apple.security.automation.apple-events` = true (for AX)
     - SUFeedURL and SUPublicEDKey (placeholders — replace later for Sparkle)

5. **Add Swift Package Dependencies (SPM)**
   - File → Add Packages…
   - Add these four (exact versions from our `mac/Package.swift`):
     1. `https://github.com/groue/GRDB.swift`  (≥6.0)
     2. `https://github.com/SwiftyJSON/SwiftyJSON` (≥5.0)
     3. `https://github.com/sparkle-project/Sparkle` (≥2.0)
     4. `https://github.com/armadsen/ORSSerialPort` (≥2.1)
   - Add them to your **ScanToExternApp** target.

6. **Capabilities & Signing (very important)**
   - Select the target → **Signing & Capabilities** tab.
   - **Team**: choose your Developer ID (RBRX2Y72NR or your own).
   - **Hardened Runtime**: **ON**.
   - **App Sandbox**: **OFF** (required for AXUIElement and CGEventPost; this is intentional per the architecture).
   - In the same tab, you can manually add the two usage descriptions if they didn't come from the plist.

7. **Build Settings tweaks**
   - Search for "Other Linker Flags" — usually nothing needed.
   - Make sure "Enable Hardened Runtime" is yes.
   - In "Info.plist Values" or directly in the plist file, confirm `LSUIElement = true`.

8. **Build & Run**
   - Select the **ScanToExternApp** scheme.
   - Product → Run (or ⌘R).
   - The app should appear **only** as a menubar icon (scanner symbol). No Dock icon, no main window.
   - Right-click the icon for the debug menu + "Check for Updates".

9. **First run permissions**
   - The app will show an onboarding sheet if Accessibility is missing.
   - Grant Accessibility in System Settings → Privacy & Security.
   - The "Test Scan" button (in popover or right-click menu) lets you test the full flow immediately.

## Common Issues & Fixes
- App appears in Dock: Make sure `LSUIElement` is true in the final Info.plist used by the target and that you have no other `WindowGroup` or scenes in the `@main` App.
- AX injection does nothing: Accessibility permission not granted yet. The app prompts / has the Permissions sheet.
- No Bluetooth devices: The code scans for the Nordic UART service. Use Xcode's Bluetooth Explorer (Additional Tools) to confirm your Scanmarker's service UUID if it differs.
- Sparkle "Check for Updates" does nothing: You need a real `SUFeedURL` (appcast.xml) + public Ed key generated by `generate_keys` from Sparkle. Placeholders are in the plist.
- Extension not connecting: The native app must be running (WS server on 127.0.0.1:52731). Load the `BrowserExtension/` folder as unpacked extension in Chrome.

## After you have a working .app
- For distribution: Archive → Export → Developer ID (not App Store).
- Notarize with `xcrun notarytool`.
- Staple.
- See CLAUDE.md Phase 8 for the exact xcodebuild + notary + stapler commands + DMG creation.

## Optional but nice
- Add your real App Icon images into `Assets.xcassets/AppIcon.appiconset/`.
- Later: create a Safari Web Extension target (Xcode → File → New → Target → Safari Extension) and copy the JS/HTML from `BrowserExtension/`.

You now have a fully functional ScanToExternApp v5.0 menubar companion that can:
- Receive from simulated or real Scanmarker hardware
- Show editable preview toast with configurable timeout
- Apply Vision + on-device / Claude AI
- Inject via AX or clipboard into any focused field
- Save to searchable history with re-inject
- Work with the browser extension for web apps
- Support launch at login, secure key storage, Sparkle updates, proper permissions UX
